🔧 Drivers Réseau

• bridge : Réseau privé isolé (par défaut)
• host : Utilise l'interface hôte directement
• none : Aucun réseau (isolation totale)
• overlay : Multi-hôtes (Swarm)
• macvlan : MAC addresses uniques

Volumes Docker - Gestion des Données
📚 Stratégies de Persistance

💾 Types de Volumes

🗂️ BIND MOUNTS           🗄️ VOLUMES               🏃 TMPFS
/host/path               docker volume             RAM (temporaire)
     ↓                        ↓                         ↓
/container/path       /var/lib/docker/volumes/   /container/tmpfs

✅ Contrôle total        ✅ Géré par Docker        ✅ Performance max
❌ Dépendance hôte       ❌ Moins de contrôle      ❌ Non persistant

🔄 Drivers de Volume

• local : Stockage local (par défaut)
• nfs : Network File System
• sshfs : SSH File System
• rexray : Cloud storage (AWS EBS, etc.)

Docker Compose & Stacks - Orchestration
📚 Orchestration Multi-Conteneurs

📋 Structure docker-compose.yml

version: '3.8'

services:
  web:
    image: nginx
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html
    networks:
      - app_network
    depends_on:
      - db

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: secret
    volumes:
      - mysql_data:/var/lib/mysql

volumes:
  mysql_data:

networks:
  app_network:

🎯 Concepts Clés

• Services : Définitions de conteneurs
• Networks : Réseaux personnalisés
• Volumes : Persistance des données
• depends_on : Ordre de démarrage
• Environment : Variables d'environnement

Sécurité Docker - Hardening & Best Practices
📚 Sécurité Conteneurs

🔒 Vecteurs d'Attaque

• Container escape : Sortir du namespace
• Privilege escalation : Root dans conteneur
• Image vulnerabilities : CVE dans les images
• Network exposure : Ports exposés
• Secrets exposure : Credentials dans images

🛡️ Mesures de Protection

• Non-root users : UID/GID mapping
• Read-only filesystems : Immutable containers
• Capabilities dropping : Linux capabilities
• AppArmor/SELinux : MAC policies
• Network policies : Firewall conteneurs

🔐 Security Audit
🎯 Exercise 9.1: Audit Sécurité CyberLab
Container cyberlab_dashboard → Inspect
User: identifier si root (UID 0) ou non-root
Capabilities: lister les privileges Linux
Network: ports exposés vs nécessaires

🚨 Audit Findings CyberLab

Points d'attention :

• Root containers : Plusieurs services en root
• Docker socket : Monté sur dashboard
• Network : Tous services sur même réseau
• Secrets : Passwords en variables d'env

Troubleshooting & DevOps Workflow
📚 Diagnostic Professionnel

🔍 Méthologie de Debug

1. Identifier le problème : Logs, métriques

🛠️ Outils de Debug

• docker exec : Shell interactif
• docker logs : Analyse des sorties
• docker inspect : Configuration détaillée
• docker system : État global Docker
• netstat/ss : Connectivité réseau
